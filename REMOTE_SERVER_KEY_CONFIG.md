# 远程服务器密钥验证配置指南

## 功能说明

本系统实现了远程服务器密钥验证功能，用于验证系统是否已授权。如果授权过期，系统将无法启动和登录。

## 配置方法

### 1. 在环境变量中配置

在系统环境变量中添加以下配置：

| 环境变量名 | 说明 | 示例值 |
|------------|------|--------|
| REMOTE_SERVER_KEY | 远程服务器密钥 | your_remote_server_key |
| REMOTE_SERVER_KEY_EXPIRE | 密钥过期时间（时间戳，毫秒） | 1735689600000（2025-01-01 00:00:00） |

### 2. 在 application.yml 中配置

在 `api/src/main/resources/application.yml` 文件中添加以下配置：

```yaml
REMOTE_SERVER_KEY: your_remote_server_key
REMOTE_SERVER_KEY_EXPIRE: 1735689600000
```

### 3. 如何生成时间戳

可以使用以下方法生成时间戳：

- **在线工具**：访问 https://www.timestampconvert.com/ 生成时间戳
- **Java代码**：`System.currentTimeMillis() + 365L * 24 * 60 * 60 * 1000`（生成一年后的时间戳）
- **命令行**：在Linux/Mac上执行 `date +%s` 生成当前时间戳（秒），然后乘以1000得到毫秒

## 验证流程

1. **系统启动时验证**：系统启动时会自动验证远程服务器密钥是否有效
2. **用户登录时验证**：用户登录时会再次验证远程服务器密钥是否有效

## 错误信息

- **启动时错误**：`远程服务器密钥验证失败或已过期，系统将关闭！`
- **登录时错误**：`远程服务器密钥验证失败或已过期，无法登录！`

## 注意事项

1. 请确保配置的时间戳是有效的，并且在未来的时间
2. 密钥和过期时间请妥善保管，不要泄露给未授权的人员
3. 当密钥过期时，需要重新配置新的密钥和过期时间

## 测试方法

1. 配置有效的远程服务器密钥和过期时间
2. 启动系统，应该能正常启动
3. 尝试登录系统，应该能正常登录
4. 修改过期时间为过去的时间，重启系统，应该会看到错误信息
5. 移除密钥配置，重启系统，应该会看到错误信息
